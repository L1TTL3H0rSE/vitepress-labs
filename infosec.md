# ОТЧЕТ ПО ЛАБОРАТОРНЫМ РАБОТАМ

**По дисциплине:** «Информационная безопасность и защита информации»

## Введение

В ходе выполнения цикла лабораторных работ было разработано единое мобильное приложение на фреймворке **Flutter (Dart)**. Приложение демонстрирует применение различных механизмов обеспечения информационной безопасности: от классических криптографических алгоритмов до современных методов аутентификации (OAuth 2.0, биометрия) и безопасного хранения учетных данных.

## Лабораторная работа №1 и №2. Шифры Атбаш и Цезаря

**Цель:** Реализовать алгоритмы зашифровки и дешифровки текста классическими методами (Атбаш и Цезарь) с поддержкой русского и английского алфавитов.

**Ход работы:**
Разработан пользовательский интерфейс с использованием вкладок (`TabBar`), позволяющий переключаться между шифрами. Для выбора языка используется виджет `SegmentedButton`. 

Алгоритмы реализованы в классе `Lab1Utils`:

1. **Шифр Атбаш:** Заменяет $i$-ю букву алфавита на $(N-i-1)$-ю, где $N$ — длина алфавита. Приложение корректно обрабатывает регистр символов и игнорирует знаки препинания.
2. **Шифр Цезаря:** Осуществляет циклический сдвиг символов на заданное число позиций. Для дешифровки применяется обратный сдвиг (`shift = -shift`). Для циклического сдвига используется модульная арифметика: `(idx + shift) % alphabet.length`.

## Лабораторная работа №3. OAuth 2.0. Интеграция авторизации

**Цель:** Внедрить авторизацию с помощью стороннего отечественного сервиса в соответствии с ФЗ № 406.

**Ход работы:**
В качестве провайдера авторизации выбран сервис **ВКонтакте (VK)**. В связи с ограничениями на использование сторонних плагинов, алгоритм OAuth 2.0 (Implicit Flow) реализован самостоятельно с помощью `webview_flutter`.

**Алгоритм работы:**

1. Приложение открывает страницу авторизации VK через `WebViewController`.
2. В параметрах передаются `client_id`, `redirect_uri=blank.html` и `response_type=token`.
3. С помощью `NavigationDelegate` перехватывается URL перенаправления.
4. Если URL содержит `access_token`, он извлекается с помощью `Uri.queryParameters`, WebView закрывается, а пользователь получает доступ в систему.

## Лабораторная работа №5 и №6. Интерфейс авторизации и безопасное хранение паролей

**Цель:** Создать защищенную систему регистрации и авторизации пользователей, реализовать валидацию полей и криптостойкое хранение паролей.

**Ход работы:**
Разработаны экраны входа и регистрации. Особое внимание уделено защите данных:

1. **Валидация (Лаб. 5):** В поле пароля используется регулярное выражение `r'^(?=.*?[0-9])(?=.*?[!@#\$&*~]).{8,}$'`, которое требует минимум 8 символов, наличие цифры и спецсимвола. Реализована проверка на совпадение паролей в полях «Пароль» и «Подтвердите пароль». Символы пароля скрываются при вводе (`obscureText: true`).
2. **Безопасное хранение (Лаб. 6):** Для хранения данных используется локальная база данных SQLite (`sqflite`).
   Вместо хранения паролей в открытом виде используется алгоритм **bcrypt**. При регистрации вызывается функция `BCrypt.hashpw()`, которая генерирует соль и хеширует пароль. В базу данных (`users(login, hash)`) сохраняется только полученный хеш. При авторизации введенный пароль проверяется функцией `BCrypt.checkpw()`, которая сравнивает хеши, что делает невозможным восстановление пароля даже при компрометации базы данных.

## Лабораторная работа №7. Аудит разрешений мобильного приложения

**Цель:** Изучить и реализовать управление правами доступа приложения к чувствительным данным устройства.

**Ход работы:**
В `AndroidManifest.xml` задекларированы следующие разрешения: `CAMERA`, `INTERNET`, `RECORD_AUDIO`, `ACCESS_FINE_LOCATION`, `USE_BIOMETRIC`.

Разработан экран аудита на базе пакета `permission_handler`. Для каждого чувствительного разрешения определена и описана бизнес-цель:

* **Камера:** необходима для создания фото профиля и сканирования QR-кодов.
* **Микрофон:** для голосового управления/ввода.
* **Геолокация:** для отслеживания подозрительных попыток входа (гео-трекинг сессий).
* **Биометрия:** для бекпарольного входа в систему.

Интерфейс приложения позволяет просматривать текущий статус каждого разрешения (выдано/отклонено/заблокировано) и запрашивать их в реальном времени у пользователя операционной системы.
