# Курсовая работа: Создание языка программирования

**Дисциплина:** «Лингвистическое обеспечение САПР»  
**Задание №:** 18

## 1. Постановка задачи

В данной курсовой работе необходимо разработать собственный язык программирования и транслирующий компилятор.
Компилятор должен переводить исходный текст программы на разработанном языке в эквивалентную программу на языке высокого уровня (в данном проекте целевым языком выступает **JavaScript**) для её последующего безошибочного выполнения.

**Требования к языку:**

- Относится к языкам высокого уровня, поддерживает структурное программирование.
- Ключевые слова имеют русские названия. Идентификаторы состоят из латинских или русских букв.
- **Базовые типы данных:** длинное целое положительное число (`целое`); квадратная матрица целых чисел (`матрица`).
- **Операции:** сложение (`+`), вычитание (`-`), сравнения (`>`, `<`, `==`, `!=`).
- **Операторы:** ввод и вывод, цикл с предусловием (`пока`), условный оператор (`если ... иначе`), сортировка строк матрицы (`сортировать`).

**Требования к компилятору:**
Представить в виде интегрированной оболочки с выводом соответствующих сообщений при наличии синтаксических или лексических ошибок.

## 2. Теоретические основы создания компиляторов

Архитектура разработанного компилятора построена по классической многопроходной модели и состоит из трех основных модулей:

1. **Лексический анализатор (Сканер / Лексер).**
   Читает исходный текст программы посимвольно и группирует символы в лексемы (токены). Лексер удаляет пробельные символы и классифицирует слова языка (Ключевое слово, Идентификатор, Число, Оператор, Пунктуация). Анализатор реализован в виде конечного автомата.
2. **Синтаксический анализатор (Парсер).**
   Принимает на вход массив токенов и проверяет их на соответствие грамматике языка. Реализован с использованием алгоритма **рекурсивного спуска (Recursive Descent)**. Результатом работы парсера является Абстрактное Синтаксическое Дерево (AST) — иерархическая структура объектов, отражающая семантику программы. Учитывает приоритет операций (сначала сравнения, затем сложение/вычитание).
3. **Генератор кода (Транслятор).**
   Обходит полученное AST-дерево с использованием паттерна проектирования **Visitor (Посетитель)**. Для каждого узла дерева генерируется семантически эквивалентный код на целевом языке (JavaScript).

## 3. Описание синтаксиса языка

Синтаксис языка основан на С-подобной грамматике с использованием фигурных скобок для выделения блоков кода. Каждая инструкция завершается точкой с запятой `;`.

- **Объявление переменных:**
    `целое <имя>;`
    `матрица <имя>[<размер>];`
- **Присваивание и математика:**
    `размер = размер + 1;`
- **Условный оператор:**
    `если (<условие>) { ... } иначе { ... }`
- **Цикл с предусловием:**
    `пока (<условие>) { ... }`
- **Встроенные операторы:**
    `ввод(<имя>);`, `вывод(<имя>);`, `сортировать(<имя>);`

## 4. Описание алгоритмов трансляции

Трансляция исходного кода (AST) в JavaScript выполняется по следующим правилам:

1. **Инициализация переменных:**
   - Узел `целое a;` транслируется в `let a = 0;`.
   - Узел `матрица М[3];` транслируется в создание двумерного массива: `let М = Array(3).fill().map(() => Array(3).fill(0));`.
2. **Встроенные функции и Runtime Environment:**
   Перед основным сгенерированным кодом транслятор вставляет служебные функции:
   - `ввод(X)` заменяется на вызов обертки `X = __input("X")`, которая использует системный ввод (prompt).
   - `вывод(X)` заменяется на стандартный метод `console.log(X);`.
   - `сортировать(M)` заменяется на вызов сгенерированной функции `__sortMatrix(M)`, которая проходит по строкам матрицы и вызывает встроенный алгоритм сортировки массивов `sort((a,b) => a - b)`.
3. **Блочные конструкции:**
   Узлы `WhileStatement` и `IfStatement` рекурсивно обходят свои внутренние массивы инструкций (`body` / `consequent`), инкрементируя переменную уровня отступа (`indentLevel`) для красивого форматирования результирующего кода.

## 5. Текст тестовой программы

```text
целое размер;
матрица М[3];

размер = 3;
пока (размер > 0) {
    если (размер == 2) {
        сортировать(М);
    } иначе {
        вывод(размер);
    }
    размер = размер - 1;
}
```

## 7. Список сообщений об ошибках

Компилятор обеспечивает отлов лексических и синтаксических ошибок с указанием точной строки и символа (позиции в коде).

**Лексические ошибки:**

- `Лексическая ошибка: неизвестный символ '{char}' на строке {line}, позиция {column}` (Возникает при встрече символов, не входящих в алфавит языка, например, `@` или `#`).

**Синтаксические ошибки (вызываются методом `expect`):**

- `Синтаксическая ошибка [строка X]: Ожидалось имя переменной` (После ключевого слова типа данных отсутствует идентификатор).
- `Синтаксическая ошибка [строка X]: Ожидалась ';' в конце строки`
- `Синтаксическая ошибка [строка X]: Ожидалось выражение (число или переменная)` (Ошибка в правой части математического выражения).
- `Синтаксическая ошибка [строка X]: Ожидалось '(' после имени функции`
- `Синтаксическая ошибка [строка X]: Неизвестная инструкция '...'` (Нарушение структуры программы).
