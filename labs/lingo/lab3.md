# Практическая задача №3: Синтаксический анализ выражений

Данный раздел посвящен **Лабораторной работе №7**. Эта задача является логическим развитием алгоритма, написанного в рамках Практической работы №2 (Стековый калькулятор и алгоритм сортировочной станции).

Основное отличие лабораторной работы заключается в **строгом лексическом анализе (валидации)** пользовательского ввода и поддержке математических функций (в частности, тригонометрической функции `sin`).

## Лабораторная работа №7. Разработка строчного калькулятора

**Задание:** Вычислить значение арифметического выражения, заданного в виде строки. При вычислении необходимо учитывать приоритет операций (`+`, `-`, `*`, `/`, `^`), правильность расстановки скобок, а также осуществлять валидацию введенных данных.

**Примеры тестовых случаев (скопируйте их в поле ниже для проверки):**

* `sin(sin(0.2)+0.1)/2` — Корректный ввод. Выведет `0.15` (с точностью до 2-х знаков).
* `sin(sin(0.2)+0.1)(4-2)` — Ошибка: отсутствует оператор между скобками `)(`.
* `sin(sin(0.2)+0.1)/*(4-2)` — Ошибка: два арифметических оператора подряд `/*`.
* `sin(sin(0.2)0.1)*(4-2)` — Ошибка: нет оператора между скобкой и числом `)0.1`.
* `sin(sin(0.2+0.1)*(4-2)` — Ошибка: не хватает закрывающей скобки.
* `sinn(sin(0.2)+0.1)*(4-2)` — Ошибка: неизвестное имя функции `sinn`.

## Архитектура решения

Работа алгоритма разбита на три классические фазы трансляции:

1. **Лексический анализ и Валидация (Scanner/Validator):**
   Регулярное выражение `/(?:[a-zA-Z]+|\d+\.\d+|\d+|[+\-*/^()])/g` разбивает строку на токены. Далее запускается цикл, который анализирует контекст токена:
   * Проверяет неизвестные буквенные последовательности (разрешено только `sin`).
   * Считает баланс скобок (`openParens`).
   * Заглядывает в предыдущий токен (`prev`), чтобы отловить ошибки вида `)(`, `)число` или `++`. При нахождении ошибки выбрасывается исключение с детальным описанием (по требованиям лабораторной работы).

2. **Синтаксический анализ (Parser):**
   Для учета приоритета операций и вложенности скобок используется **Алгоритм сортировочной станции (Shunting-yard algorithm)** Э. Дейкстры. Он переводит классическую инфиксную запись (со скобками) в обратную польскую запись (ОПЗ, Постфикс). Функция `sin` помещается в стек операций наравне с операторами, но с наивысшим приоритетом.

3. **Генерация результата (Evaluator):**
   Массив постфиксной записи вычисляется с помощью стековой машины. Операторы снимают со стека два числа, функция `sin` снимает со стека одно число, вычисляет радианный синус (`Math.sin()`) и кладет результат обратно. Итоговый ответ форматируется до двух знаков: `result.toFixed(2)`.
